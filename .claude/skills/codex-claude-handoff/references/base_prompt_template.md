# Codex -> Claude 修正依頼テンプレート

## 1. ベーステンプレート
```markdown
あなたはこのリポジトリの実装担当です。以下の修正を実施してください。

## 目的
<何を解決したいかを1文で>

## 現状
- <現状の挙動1>
- <現状の挙動2>

## 期待挙動
- <期待する挙動1>
- <期待する挙動2>

## 対象ファイル
- `<path/to/file1>`
- `<path/to/file2>`

## 根拠情報
- <再現手順>
- <ログ抜粋 or エラーメッセージ>

## 制約
- <必須制約1>
- <必須制約2>

## 非対象
- <今回変更しない範囲>

## 受け入れ条件
- [ ] <条件1>
- [ ] <条件2>

## 実行してほしい確認
- `<command 1>`
- `<command 2>`

## 返答形式
1. 変更概要（3行以内）
2. 変更ファイル一覧
3. 検証結果
4. 残課題（あれば）
```

## 2. PRレビュー指摘対応テンプレート
```markdown
以下のレビュー指摘を実装で解消してください。

## 指摘内容
- <review comment>

## 修正方針
- <どう直すか>

## 対象
- `<path/to/file>`

## 受け入れ条件
- [ ] 指摘が再現しない
- [ ] 既存テストが通る
- [ ] 変更意図をコード上で説明できる

## 実行コマンド
- `<test command>`
- `<lint command>`
```

## 3. 記入ガイド
- 目的は「手段」ではなく「解決したい問題」を書く。
- 期待挙動はテスト可能な文にする。
- 制約は優先順位順に書く。
- 非対象を書くことで過剰修正を防ぐ。
